# IpBlack API 文档

## 接口概述

IpBlack 控制器负责IP黑名单管理，支持对恶意IP的封禁和管理。

**接口类型**：基础接口

---

## 状态码规范

| 状态码 | 使用场景 | 开发注意事项 |
|--------|----------|--------------|
| 200 | 接口调用成功且数据返回正常 | 所有正常响应必须返回200 |
| 204 | 接口调用成功但无数据返回 | 空数据场景必须返回204 |
| 400 | 请求参数错误或业务校验失败 | 参数校验失败 |
| 405 | 请求方法不允许 | 调用了未定义的method |
| 500 | 服务器异常 | 生产环境屏蔽具体错误 |

---

## 接口列表

### 1. 获取指定记录 [基础接口-获取指定]

**请求方式**：GET  
**请求路径**：`/api/ip-black/one`

**请求参数**：

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| id | int | 是 | 记录ID |
| field | string | 否 | 字段过滤 |

**响应示例**：

```json
{
  "code": 200,
  "msg": "数据请求成功！",
  "data": {
    "id": 1,
    "ip": "192.168.1.1",
    "cause": "恶意攻击",
    "remark": "多次尝试暴力破解",
    "create_time": 1699000000
  }
}
```

### 2. 获取记录列表 [基础接口-获取全部]

**请求方式**：GET  
**请求路径**：`/api/ip-black/all`

**请求参数**：

| 参数名 | 类型 | 必填 | 默认值 | 说明 |
|--------|------|------|--------|------|
| page | int | 否 | 1 | 页码 |
| field | string | 否 | - | 字段过滤 |
| where | string | 否 | - | 条件查询 |
| like | string | 否 | - | 模糊查询 |
| order | string | 否 | create_time desc | 排序 |
| onlyTrashed | bool | 否 | false | 只查询已删除 |
| withTrashed | bool | 否 | false | 包含已删除 |

**响应示例**：

```json
{
  "code": 200,
  "msg": "数据请求成功！",
  "data": {
    "data": [...],
    "count": 10,
    "page": 1
  }
}
```

## 特殊说明

### IP格式处理

- IP地址自动转换为大写格式存储
- 支持IPv4地址格式

### 缓存机制

- 查询接口支持缓存
- 数据更新后自动清除相关缓存
