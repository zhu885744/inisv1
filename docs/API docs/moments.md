# Moments 接口文档

## 接口概述

`moments` 控制器用于管理动态内容（类似QQ空间动态），支持动态的创建、编辑、查询、删除等操作。包含作者信息关联、权限控制和**草稿功能**。

### 接口类型说明

| 接口类型 | 说明 |
| :--- | :--- |
| **基础接口** | 支持15个基础接口：one、all、rand、count、sum、min、max、column、remove、delete、clear、restore、save、create、update |

### 状态字段说明

| 状态值 | 说明 |
| :--- | :--- |
| **0** | 草稿 - 动态未发布，不对外展示 |
| **1** | 发布 - 动态已发布，可以正常展示 |

### 审核字段说明

| 审核值 | 说明 |
| :--- | :--- |
| **0** | 待审核 - 动态待管理员审核，普通用户不可见 |
| **1** | 已审核 - 动态已通过审核 |

### 审核状态组合

| audit | status | 含义 | 普通用户可见 |
| :--- | :--- | :--- | :--- |
| 0 | 0 | 待审核 | 否 |
| 1 | 1 | 已发布已审核 | 是 |
| 1 | 0 | 草稿 | 否 |

## 状态码规范

| 状态码 | 说明 | 使用场景 |
| :--- | :--- | :--- |
| **200** | 请求成功 | 获取数据成功、操作成功 |
| **204** | 无内容 | 查询无数据、无可操作数据 |
| **400** | 请求错误 | 参数校验失败、操作失败 |
| **401** | 未授权 | 用户未登录 |
| **403** | 无权限 | 无操作权限 |
| **405** | 方法不允许 | 请求方法错误或方法名错误 |
| **500** | 服务器错误 | 系统内部错误 |

### 2. POST 请求接口

#### 2.1 保存动态 [基础接口-保存数据]

- **路径**: `/api/moments/save`
- **方法**: `POST`
- **描述**: 保存动态（新增或更新），支持保存为草稿，支持审核功能

**请求参数**:

| 参数名 | 类型 | 必填 | 说明 |
| :--- | :--- | :--- | :--- |
| `id` | int | 否 | 动态ID，为空时新增 |
| `content` | string | **是** | 动态内容 |
| `images` | string | 否 | 图片，逗号分隔 |
| `location` | string | 否 | 位置信息 |
| `json` | json | 否 | JSON数据 |
| `text` | string | 否 | 文本内容 |
| `publish_time` | int | 否 | 发布时间戳 |
| `status` | int | 否 | 状态：0-草稿，1-发布（默认1） |
| `audit` | int | 否 | 审核状态：0-待审核，1-已审核（仅管理员可用） |

**审核逻辑**:

| 操作 | MOMENTS.audit=1（开启审核） | MOMENTS.audit=0（关闭审核） |
| :--- | :--- | :--- |
| 保存草稿 | audit=1, status=0 | audit=1, status=0 |
| 发布动态 | audit=0, status=0（待审核） | audit=1, status=1（直接发布） |

**成功响应** (200):
```json
{
    "code": 200,
    "msg": "创建成功！",
    "data": {
        "id": 1
    }
}
```

**草稿响应** (200):
```json
{
    "code": 200,
    "msg": "草稿保存成功！",
    "data": {
        "id": 1
    }
}
```

## 特殊说明

### 1. 作者信息
- 动态查询时会自动关联作者信息（用户昵称、头像等）
- 作者信息存储在 `result.author` 字段中

### 2. 图片处理
- `images` 字段支持数组格式，会自动序列化为逗号分隔的字符串

### 3. 草稿功能
- 支持保存为草稿（status=0），草稿不对外展示
- 草稿发布后（status=1），会设置发布时间

### 4. 缓存策略
- 所有查询接口均支持缓存
- 数据修改后会自动清除相关缓存

### 5. 字段处理
- `images` 支持数组格式，会自动序列化
- `json` 类型字段会自动序列化
